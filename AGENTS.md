# AGENTS.md

## 목적
이 문서는 `ttalkkakthon` 프로젝트에서 에이전트가 따라야 하는 **프로젝트 전역 동작 규칙**과 **문서 관리 규칙**을 정의한다.

## 전역 규칙 변경 반영 규칙
- 사용자 요청으로 에이전트의 프로젝트 내 전역 동작 규칙(작업 방식, 우선순위, 문서 운영 규칙, 품질 기준 등)이 변경되면, 해당 변경 사항을 **같은 작업 턴 내에** 이 `AGENTS.md`에 반영한다.
- 위 규칙은 해당 요청 자체를 포함해, 이후의 모든 요청에 동일하게 적용한다.
- 반영 시에는 아래를 명시한다.
  - 무엇이 바뀌었는지
  - 어떤 범위(전체 프로젝트/특정 문서 체계/특정 워크플로우)에 적용되는지
  - 앞으로 어떤 방식으로 동작해야 하는지

## 프로젝트 개요/기술 스택 변경 반영 규칙
- 사용자 요청으로 프로젝트의 전체 개요, 구조, 핵심 정책, 기술 스택이 변경되면 `AGENTS.md`에 현재 기준으로 업데이트한다.
- 스택 변경 예시: 프레임워크, 언어, 스타일링, 데이터베이스, ORM, 패키지 매니저, 배포/런타임 정책.

## 기능 명세 문서 규칙
- 기능 명세는 `docs/feature` 디렉토리에서 관리한다.
- 파일명은 증가하는 번호 인덱스를 사용한다.
  - 형식: `NNN-feature-name.md` (예: `001-landing-page.md`)
- 기존 기능의 동작이 수정/추가되면 **새 파일을 만들지 않고 해당 기능 파일을 갱신**한다.
- 신규 기능일 때만 다음 번호의 새 파일을 생성한다.

## 컴포넌트 명세 문서 규칙
- 컴포넌트 명세는 `docs/feature/components` 디렉토리에서 컴포넌트별 개별 파일로 관리한다.
- 컴포넌트 관련 요구사항/동작/UI/입출력(Props)이 변경되거나 추가되면, 해당 컴포넌트 문서를 **같은 작업 턴 내에** 반드시 갱신한다.
- 신규 컴포넌트가 추가되면 해당 디렉토리에 신규 명세 파일을 생성하고 인덱스(`README.md`)에 등록한다.

## 현재 프로젝트 기준 정보 (2026-03-04)
- Framework: Next.js 16 (App Router)
- Language: TypeScript (strict)
- Styling: Tailwind CSS v4
- ORM: Prisma v7
- Database: PostgreSQL (Neon/Vercel Postgres)
- Package Manager: pnpm
