# 투표 시스템 설계

## 개요

해커톤 당일 참가자 투표를 통해 프로젝트를 평가한다.
참가 신청 시 수집한 이메일/전화번호를 기반으로 투표 자격을 확인하고, 공정성을 확보한다.

## 핵심 규칙

| 항목      | 내용                                       |
| --------- | ------------------------------------------ |
| 투표 자격 | 참가 신청된 이메일 또는 전화번호로 본인 확인 |
| 투표 수   | 1인당 5~10표                               |
| 배분 규칙 | 프로젝트당 최대 1표 (5~10개 프로젝트 선택) |
| 제한      | 본인 팀 프로젝트에는 투표 불가             |
| 평가 비중 | 참가자 투표 50% + 심사위원 평가 50%        |

## 투표 흐름

```
이메일 또는 전화번호 입력 → 참가 신청 여부 확인 → 잔여 투표 수 확인 → 투표 화면 → 프로젝트 선택 → 제출
```

## 참가 신청 변경사항

현재 팀 대표자 이메일만 필수 수집 중이므로, **모든 팀원의 이메일을 필수로 변경**한다.

- 개인 참가: 기존과 동일 (이메일 필수)
- 팀 참가: 모든 팀원 이메일 필수로 변경

## 검증 방식

별도 인증(코드 발송 등) 없이 간소화한다.

1. 이메일 또는 전화번호 입력
2. DB에서 참가 신청 여부 조회 (대표자 + 팀원 모두 검색)
3. 일치하면 → 해당 인원의 팀 정보 + 잔여 투표 수 확인
4. 투표 진행

> 해커톤 현장에서 진행하므로 대리투표 리스크가 낮아 인증 생략

## 데이터 모델 (추가)

```
Vote
├── id
├── voterEmail    ← 투표자 식별 (이메일)
├── voterPhone    ← 투표자 식별 (전화번호)
├── teamId        ← 투표자 소속 팀 (본인 팀 투표 방지용)
├── projectId     ← 투표 대상 프로젝트 (FK)
├── createdAt

VoteSession
├── id
├── isActive      ← 투표 활성화 여부 (관리자 제어)
├── startedAt
├── endedAt
```

## 미결정 사항

- [ ] 투표 결과 실시간 공개 여부
- [ ] 심사위원 투표를 같은 시스템으로 처리할지
